# Security Policy

## Supported Versions

This is a dotfiles repository. There are no binary releases to patch. The latest commit on the `main` branch is the supported version.

## Reporting a Vulnerability

- Preferably create a private GitHub Security Advisory for this repository if available.
- If advisories are not enabled, open a new issue with the title prefix `[Security]` and minimal reproduction details. Avoid sharing sensitive information publicly.
- Alternatively, you can open a discussion to ask for guidance before disclosure.

We follow responsible disclosure. Maintainers will evaluate and respond within a reasonable timeframe.

## Best Practices

- Review scripts before running on your system.
- Inspect installation commands and package lists in `executable_arch-setup-script.sh`.
- Submit a PR with improvements or mitigations where applicable.
